# Quick Start Guide

## Starting the Application

1. Open a terminal in the project directory
2. Install dependencies (first time only):
   ```bash
   pip install -r requirements.txt
   ```
3. Run the app:
   ```bash
   python app.py
   ```
4. Open your browser to: **http://localhost:5000**

## Your First Experiment

### Week 1 - Establish Baseline
1. Go to **Settings** ‚Üí Set "Experiment Week" to "Week 1"
2. Go to **Hunger** ‚Üí Toggle "Baseline Mode" ON
3. Log hunger entries WITHOUT Allulose to establish your baseline
4. Log subjective ratings throughout the day

### Week 2+ - Test Allulose
1. Update to "Week 2" in Settings
2. Turn OFF Baseline Mode
3. On **Dashboard**, click "Start Timer" when consuming Allulose
4. Select Variant:
   - **Variant A**: Meal + Allulose
   - **Variant B**: Allulose only (safety reminder will appear)
5. When hungry, click "I'm Hungry - Log Entry"
6. Fill in confounding variables (sleep, stress, Concerta timing, etc.)

### Tracking Throughout the Day

**Subjective Ratings** (‚≠ê Ratings tab):
- Quick-rate Energy, Clarity, Mood, Physical performance
- Tap metric ‚Üí slide 1-10 ‚Üí submit
- View time-series chart showing daily patterns

**GI Events** (üîî GI tab):
- ONLY log when you experience discomfort
- Enter: amount, time since consumption, symptoms, severity
- App automatically calculates your tolerance threshold

## Understanding Your Data

### Dashboard Insights
- **Today's Ratings**: Average scores for each metric today
- **Baseline Avg**: Your hunger delay without Allulose
- **Allulose Avg**: Your hunger delay with Allulose
- **Improvement**: The difference (positive = Allulose is working!)
- **GI Threshold Warning**: Appears if you've logged severe events

### Syncing Between Devices
1. **Export** from Settings on Device A
2. Save JSON file to cloud storage (Dropbox, Google Drive, etc.)
3. Download on Device B
4. **Import** from Settings on Device B
